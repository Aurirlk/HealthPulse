## 四、关键架构决策

### 4.1 双 AI 体系

| | AI 医生 | CRM 前台助理 |
|------|----------|-------------|
| 接口 | `/ai/chat`、`/ai/chat/stream` | `/crm/chat`、`/crm/chat/stream` |
| 认证 | JWT（`@Protector`） | 无（手机号标识） |
| 用户标识 | userId（JWT 中） | phoneNumber（请求体） |
| 健康数据 | 对话前注入 system prompt | 通过 `GetHealthDataTool` function calling |
| 知识库 RAG | 关键词 LIKE 检索 news 表 | 向量语义搜索 + Keyword |
| 数据库 | MySQL | SQLite |
| 技术范式 | 传统 System Prompt + 上下文注入 | ReAct Agent + Function Calling |

### 4.2 ReAct Agent 循环

```
用户消息 → buildInitialMessages → [System Prompt + 历史 + 用户消息]
   ↓
callLLMWithTools(messages)  ←──┐
   ↓                            │
有 tool_calls? ─Yes→ 执行工具 ──┘
   │No                          (最多 5 轮)
   ↓
streamFinalAnswer → SSE 逐 token 推送
```

### 4.3 向量数据库存储格式

```
vectors.bin:  [f32*1536][f32*1536]...  (固定长度记录，无 header)
metadata.json: 集合元数据
documents.json: 文档内容 + metadata
collections.json: 全局注册表
```

启动时全部加载到 `ArrayList<float[]>`，搜索用余弦相似度 + PriorityQueue TopK。

### 4.4 为什么 SQLite？

- CRM 用手机号标识，不走 MySQL 用户体系
- 嵌入式数据库，零外部依赖，适合 SQL Agent 演示
- 通过 `SQLiteConfig.setReadOnly(true)` 实现内核级只读防护
- 读写连接分离，只读查询有 maxRows(200) + queryTimeout(10s)

### 4.5 System Prompt 外置

`application.yml` 中 `crm.react.prompt` 可覆盖默认 Prompt，无需重新编译。为空则用 `BaseReActAgent` 内置默认值。

## 七、已知注意事项

1. **API Key 必须注入**：`application.yml` 中 DeepSeek API Key 已清空，启动前必须设 `DEEPSEEK_API_KEY` 环境变量

2. **JWT 豁免**：`/crm/**` 路径已在 `InterceptorConfig` 中排除 JWT 校验

3. **CRM 首次启动**：自动创建 `crm_data/` 目录（含 SQLite + 向量库）

4. **MySQL 表结构**：`sql/mysql_schema.sql` 有完整建表语句，10 张表带外键

5. **Scala**：`FileController.sanitizeFileName` 已修复吞扩展名 Bug（`\\.` → `\\.{2,}`）

6. **资讯封面图**：`NewsServiceImpl` 有启动时扫描 pic/ 目录的兜底逻辑，cover 为空时自动分配

7. **RAG 关键词提取**：`AiServiceImpl.extractKeyword()` 智能截取用户消息最后一句作为检索关键词

8. **编译警告**：Java 8 不支持 `var`/`Map.of()`/`List.of()`，代码已全部兼容

9. **`@Protector(role = "管理员")`** 中的 "管理员" 是中文 String，来自 `RoleEnum`

10. **`AiServiceImpl.buildArticleRagContext` 限流**：每篇摘要截取 300 字符，每次最多返回 3 篇
